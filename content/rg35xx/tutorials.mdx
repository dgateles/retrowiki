O RG35XX não tem Wi-Fi nem Bluetooth. Toda transferência de arquivos — ROMs, BIOS, saves, temas — é feita pelo cartão microSD ou pelo cabo USB-C. Abaixo estão os fluxos mais comuns.

---

## Adicionar ROMs

### Via cartão microSD

<Steps>
  <Step title="Desligue o console">
    Segure o botão Power e confirme o desligamento no firmware, ou aguarde o console hibernar.
  </Step>
  <Step title="Remova o cartão de ROMs">
    Retire o cartão do slot **TF2** (ou TF1, se usar cartão único) e insira no leitor de cartão do PC.
  </Step>
  <Step title="Copie as ROMs para as pastas corretas">
    Navegue até a pasta de ROMs do cartão e copie os arquivos na subpasta do sistema:

    | Sistema          | Pasta (GarlicOS)          |
    | ---------------- | ------------------------- |
    | Game Boy / GBC   | `/Roms/GB/` ou `/Roms/GBC/` |
    | Game Boy Advance | `/Roms/GBA/`              |
    | NES / Famicom    | `/Roms/FC/`               |
    | SNES             | `/Roms/SFC/`              |
    | Mega Drive       | `/Roms/MD/`               |
    | PlayStation 1    | `/Roms/PS/`               |
    | Nintendo 64      | `/Roms/N64/`              |
    | Nintendo DS      | `/Roms/NDS/`              |
    | PSP              | `/Roms/PSP/`              |

    No muOS use `/ROMS/[SISTEMA]/`; no KNULLI use `userdata/roms/[SISTEMA]/`. Consulte a [página de Cartão SD](/rg35xx/sd-card) para a tabela completa por firmware.
  </Step>
  <Step title="Reinsira e ligue">
    Coloque o cartão de volta no console e ligue. O firmware varre os novos arquivos automaticamente no boot.
  </Step>
</Steps>

### Via cabo USB-C

Alguns firmwares permitem montar o cartão como unidade USB sem removê-lo:

<Steps>
  <Step title="Conecte o USB-C ao PC">
    Ligue o cabo com o console ligado.
  </Step>
  <Step title="Ative o modo de armazenamento">
    No muOS: **Aplicativos → Transferência USB**. No KNULLI: **Configurações → Serviços → USB Storage**.
  </Step>
  <Step title="Copie e ejete com segurança">
    O cartão aparece como drive no PC. Copie os arquivos, ejete pelo sistema operacional e reinicie o console.
  </Step>
</Steps>

<Alert type="warning" title="GarlicOS não tem transferência USB">
  O GarlicOS original não suporta modo de armazenamento USB. Para esse firmware, use sempre o método do cartão físico.
</Alert>

## Instalar Arquivos BIOS

PS1, Nintendo DS, Neo Geo e alguns outros sistemas exigem BIOS para emular corretamente.

<Steps>
  <Step title="Obtenha os arquivos BIOS">
    Os nomes devem ser exatos, incluindo maiúsculas e minúsculas. Exemplos dos mais usados:

    | Sistema         | Arquivo de BIOS                          |
    | --------------- | ---------------------------------------- |
    | PlayStation 1   | `scph1001.bin`                           |
    | Nintendo DS     | `bios7.bin`, `bios9.bin`, `firmware.bin` |
    | Neo Geo         | `neogeo.zip`                             |
    | Mega CD / Sega CD | `bios_CD_U.bin`                        |
  </Step>
  <Step title="Copie para a pasta correta do firmware">
    | Firmware  | Pasta de BIOS    |
    | --------- | ---------------- |
    | GarlicOS  | `/Bios/`         |
    | muOS      | `/MUOS/bios/`    |
    | KNULLI    | `userdata/bios/` |
    | MinUI     | `/BIOS/`         |
  </Step>
  <Step title="Verifique no RetroArch">
    Abra o RetroArch → **Informações → Verificação de BIOS** para confirmar quais arquivos foram reconhecidos corretamente.
  </Step>
</Steps>

## Configurar o RetroArch

O RetroArch é o emulador principal em todos os firmwares do RG35XX.

### Abrir o menu durante um jogo

- **muOS / KNULLI**: <kbd>MENU</kbd> + <kbd>X</kbd>
- **GarlicOS**: Segure <kbd>MENU</kbd> para ver os atalhos disponíveis na tela

### Proporção de tela

A tela do RG35XX é 4:3 nativa — configure o RetroArch para aproveitar ao máximo:

1. Abra o Quick Menu durante o jogo
2. **Quick Menu → Opções de Vídeo** (ou **Configurações → Vídeo → Escalonamento**)
3. Proporção de aspecto: **4:3** — Escalonamento inteiro: **Ativo** (pixels limpos em qualquer sistema retrô)

### Save States

| Ação           | Atalho (muOS / KNULLI)          |
| -------------- | -------------------------------- |
| Salvar estado  | <kbd>MENU</kbd> + <kbd>R2</kbd> |
| Carregar estado| <kbd>MENU</kbd> + <kbd>L2</kbd> |
| Mudar slot     | Quick Menu → State Slot          |

### Shaders (filtros visuais)

1. Quick Menu → Shaders → Carregar Shader
2. Boas opções para o RG35XX: `crt-pi` (efeito CRT), `sharp-bilinear` (pixel limpo)
3. Salve com **Salvar Preset do Jogo** para manter o filtro automaticamente

Shaders pesados podem reduzir o desempenho em SNES e GBA. Se notar lentidão, desative.

## Adicionar Capas de Jogos (Box Art)

Sem Wi-Fi, o scraping é feito no PC antes de levar o cartão para o console.

<Steps>
  <Step title="Instale o Skraper no PC">
    Baixe o [Skraper](https://www.skraper.net/) — gratuito para Windows e Mac.
  </Step>
  <Step title="Configure a conta Screenscraper">
    Crie uma conta gratuita no [Screenscraper.fr](https://www.screenscraper.fr/) e configure as credenciais no Skraper.
  </Step>
  <Step title="Aponte para a pasta de ROMs">
    Com o cartão TF2 inserido no PC, selecione o sistema no Skraper e aponte para a pasta de ROMs correspondente.
  </Step>
  <Step title="Execute o scraping e leve para o console">
    Clique em **Scrape**, aguarde o download das capas e metadados, ejete o cartão e reinsira no RG35XX.
  </Step>
</Steps>

## Atualizar o Firmware

Sem Wi-Fi, atualizações são sempre manuais:

<Steps>
  <Step title="Baixe a nova versão no PC">
    Acesse o site ou GitHub do firmware (muOS, KNULLI, GarlicOS) e baixe a imagem mais recente.
  </Step>
  <Step title="Preserve os saves">
    Se os saves estão no TF1, copie a pasta para o PC antes de regravar. Se usa dois cartões com dados no TF2, os saves são preservados automaticamente.
  </Step>
  <Step title="Grave no cartão TF1">
    Use o **Balena Etcher** ou **Rufus** para gravar a nova imagem no cartão do sistema.
  </Step>
  <Step title="Primeira inicialização">
    Insira o TF1 atualizado, o TF2 com as ROMs e ligue. Aguarde de **3 a 10 minutos** sem desligar — o sistema está expandindo partições e gerando configurações.
  </Step>
</Steps>
