O RPMini V2 chega pronto para usar — Android 13 instalado, 128 GB de armazenamento UFS 3.1 interno e Wi-Fi 6 funcionando na caixa. Os primeiros 30 minutos são de configuração; depois disso, é só jogar.

## Configuração Inicial do Android

Ao ligar pela primeira vez, o console passa por um assistente de configuração padrão do Android. Algumas etapas importantes:

1. **Pule** a configuração de conta Google se não quiser vincular agora — dá para fazer depois.
2. Vá em **Configurações → Sobre o dispositivo** e toque em **Número da compilação** sete vezes para ativar as **Opções do desenvolvedor**.
3. Em **Opções do desenvolvedor**, ative **Depuração USB** — necessário para sideload de APKs via ADB se precisar.
4. Ajuste o **modo de desempenho**: muitos apps de emulação se beneficiam do perfil de energia "Desempenho". Acesse **Configurações → Bateria → Modo de desempenho**.

### Atualizando o firmware

Antes de instalar qualquer emulador, verifique se há atualizações:

- **Configurações → Sistema → Atualização do sistema**

A Retroid costuma soltar patches nas primeiras semanas após o lançamento. Manter atualizado evita bugs de compatibilidade com os emuladores standalone.

## Instalando Emuladores

O RPMini V2 roda Android 13, então você tem acesso à Play Store. Os emuladores recomendados estão documentados em detalhes na página de **Emuladores e Frontends**. Para começar:

| Prioridade | App | Onde baixar |
|------------|-----|-------------|
| Essencial | RetroArch | Play Store / F-Droid |
| Recomendado | Dolphin Emulator | Play Store (versão oficial) |
| Recomendado | PPSSPP | Play Store |
| Recomendado | AetherSX2 / NetherSX2 | APK (sideload) |
| Recomendado | Citra (3DS) | Play Store |
| Opcional | Daijisho (frontend) | Play Store |

<Alert type="info" title="Sideload de APKs">
  Alguns emuladores como o NetherSX2 (fork do AetherSX2) são distribuídos como APK fora da Play Store. Para instalar, ative **Instalar apps de fontes desconhecidas** em Configurações → Segurança para o aplicativo de arquivos ou navegador que você usar.
</Alert>

## Onde Colocar as ROMs

Com o armazenamento interno de 128 GB, dá para manter uma biblioteca considerável no próprio console. Para expandir, use um cartão microSD (veja a página de **Cartão SD** para recomendações).

### Estrutura de pastas recomendada

```
/storage/emulated/0/
└── RetroArch/
    └── roms/
        ├── nes/
        ├── snes/
        ├── gba/
        ├── ps1/
        ├── n64/
        ├── psp/
        ├── nds/
        ├── gamecube/
        └── wii/
```

Se usar um frontend como Daijisho ou ES-DE, configure o caminho das ROMs durante o setup do frontend — eles detectam automaticamente os sistemas a partir da estrutura de pastas.

### Transferindo arquivos para o console

**Via USB (mais rápido):**
1. Conecte o RPMini V2 ao PC com cabo USB-C.
2. No console, deslize a barra de notificações e selecione **Transferência de arquivos (MTP)**.
3. No PC, o console aparece como dispositivo de armazenamento no Explorer/Finder.

**Via Wi-Fi (mais prático):**
- Instale o **Wifi FTP Server** (Play Store) ou **LocalSend** para transferência sem fio na mesma rede.
- O RetroArch também tem um servidor HTTP integrado: Menu → Configurações → Serviços → HTTP Server.

## BIOS Necessárias

Alguns sistemas exigem arquivos de BIOS para funcionar corretamente. Os principais:

| Sistema | Arquivo | Onde colocar |
|---------|---------|--------------|
| PS1 | `scph1001.bin` ou `scph5501.bin` | `/RetroArch/system/` |
| PS2 | `ps2-0200j-20020207.bin` (ou similar) | Pasta do AetherSX2/NetherSX2 |
| Dreamcast | `dc_boot.bin`, `dc_flash.bin` | `/RetroArch/system/dc/` |
| Saturn | `sega_101.bin` e `mpr-17933.bin` | `/RetroArch/system/` |
| Nintendo DS | `bios7.bin`, `bios9.bin`, `firmware.bin` | `/RetroArch/system/` |
| Game Boy Advance | `gba_bios.bin` (opcional) | `/RetroArch/system/` |

<Alert type="warning" title="Aviso Legal">
  As BIOS são protegidas por direitos autorais. Você deve extraí-las dos seus próprios consoles. Não disponibilizamos links para download de BIOS.
</Alert>

## Configurações de Tela Recomendadas

A tela AMOLED de 3.92" com resolução 1240x1080 e proporção 31:27 tem características específicas que valem ajuste:

- **Brilho automático**: desative para sessões de emulação — o brilho automático pode ser distrativo durante jogos escuros.
- **Modo escuro do Android**: ative em Configurações → Tela → Escuro. O AMOLED apaga os pixels pretos, economizando bateria.
- **Proporção no RetroArch**: para sistemas 4:3 como SNES, PS1 e N64, use a opção "Core Provided" ou "4:3" no menu de vídeo. A resolução nativa da tela é quase quadrada, então jogos 4:3 ficam excelentes sem distorção.
