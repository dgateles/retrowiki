## Adicionar ROMs ao Cartão SD

<Steps>
  <Step title="Conecte o cartão SD ao computador">
    Desligue o console antes de remover o cartão. Use um leitor de cartão de qualidade — leitores baratos podem causar erros de escrita silenciosos.
  </Step>
  <Step title="Localize a pasta Roms">
    Navegue até a pasta `Roms/` na raiz do cartão SD. Se não existir, crie-a manualmente.
  </Step>
  <Step title="Crie subpastas por sistema">
    Cada sistema precisa de uma subpasta com nome exato reconhecido pelo firmware:

    | Sistema             | Nome da pasta (NextUI / MinUI)        |
    | ------------------- | ------------------------------------- |
    | Game Boy            | `Game Boy`                            |
    | Game Boy Color      | `Game Boy Color`                      |
    | Game Boy Advance    | `Game Boy Advance` ou `GBA`           |
    | NES / Famicom       | `Nintendo Entertainment System`       |
    | SNES                | `Super Nintendo Entertainment System` |
    | Mega Drive          | `Sega Mega Drive`                     |
    | PlayStation         | `PS`                                  |
    | Nintendo DS         | `Nintendo DS (NDS)`                   |
    | Nintendo 64         | `Nintendo 64 (N64)`                   |
    | PSP                 | `PSP`                                 |
    | Neo Geo             | `Neo Geo`                             |

    O NDS merece destaque: a tela de 1024x768 (~400 PPI) do Brick reproduz os gráficos de DS com nitidez que a maioria dos handhelds retrô não consegue.
  </Step>
  <Step title="Copie as ROMs para as pastas corretas">
    Formatos aceitos variam por sistema. Os mais comuns: `.zip`, `.nes`, `.sfc`, `.gba`, `.gb`, `.gbc`, `.md`, `.bin`, `.cue`, `.iso`, `.nds`.
  </Step>
  <Step title="Atualize a lista de jogos no firmware">
    Insira o cartão e ligue o console. No NextUI e MinUI, pressione **MENU** e selecione **Refresh ROMs**. Os jogos aparecem nos sistemas correspondentes.
  </Step>
</Steps>

Para controlar a ordem de exibição no MinUI e NextUI, adicione número no início do nome da pasta: `01) Game Boy`, `02) Game Boy Color`. Os sistemas mais usados ficam no topo.

## Instalar Arquivos de BIOS

PS1, Neo Geo, NDS e GBA precisam de BIOS para funcionar corretamente:

<Steps>
  <Step title="Localize a pasta BIOS">
    Na raiz do cartão SD, localize (ou crie) a pasta `BIOS/`.
  </Step>
  <Step title="Crie subpastas por sistema">
    Dentro de `BIOS/`, uma subpasta por sistema. Exemplo: `BIOS/PS/` para PlayStation.
  </Step>
  <Step title="Copie os arquivos com o nome exato">
    Os nomes dos arquivos de BIOS são sensíveis a maiúsculas e minúsculas. Um único caractere errado faz o emulador ignorar o arquivo.

    | Sistema          | Arquivo de BIOS                         | Pasta           |
    | ---------------- | --------------------------------------- | --------------- |
    | PlayStation 1    | `scph1001.bin`                          | `BIOS/PS/`      |
    | Game Boy Advance | `gba_bios.bin`                          | `BIOS/GBA/`     |
    | Nintendo DS      | `bios7.bin`, `bios9.bin`, `firmware.bin`| `BIOS/NDS/`     |
    | Neo Geo          | `neogeo.zip`                            | `BIOS/NEOGEO/`  |
  </Step>
</Steps>

No RetroArch (KNULLI e muOS), os arquivos de BIOS ficam em `RetroArch/.retroarch/system/` em vez da pasta `BIOS/` acima.

## Configurar Wi-Fi

O Brick tem Wi-Fi 802.11 b/g/n integrado. O processo varia por firmware:

### NextUI

<Steps>
  <Step title="Acesse as configurações de rede">
    No menu principal, vá em **Configurações → Rede / Wi-Fi**.
  </Step>
  <Step title="Habilite o Wi-Fi e selecione a rede">
    Ative o Wi-Fi e aguarde as redes aparecerem. Se a lista ficar vazia, saia completamente das configurações e entre de novo — isso resolve na maioria dos casos.
  </Step>
  <Step title="Digite a senha">
    Selecione a rede e insira a senha pelo teclado virtual.
  </Step>
</Steps>

### MinUI

No MinUI, as configurações de Wi-Fi ficam na **memória interna** do console, não no cartão SD. Se você já configurou Wi-Fi no firmware de fábrica, o MinUI reconhece automaticamente. Caso contrário:

1. Acesse **Menu → Configurações → Wi-Fi**.
2. Selecione a rede e insira a senha.
3. O MinUI se reconecta nas próximas inicializações sem precisar configurar de novo.

<Alert type="info" title="Para que serve o Wi-Fi">
  RetroAchievements, scraping de capas, transferência de arquivos via rede e instalação de emuladores pela Pak Store (NextUI) — tudo depende do Wi-Fi. Para jogar offline, não é necessário.
</Alert>

## Usar RetroAchievements

RetroAchievements adiciona conquistas no estilo troféu para jogos clássicos — funciona no NextUI e muOS.

<Steps>
  <Step title="Crie uma conta">
    Acesse [retroachievements.org](https://retroachievements.org) e registre-se gratuitamente.
  </Step>
  <Step title="Configure no firmware">
    No **NextUI** ou **muOS**, acesse **Configurações → RetroAchievements** e insira usuário e senha.
  </Step>
  <Step title="Conecte ao Wi-Fi antes de jogar">
    As conquistas só aparecem com conexão ativa. Inicie o jogo com Wi-Fi conectado.
  </Step>
</Steps>

RetroAchievements funciona apenas com ROMs de hashes verificados. Se as conquistas não aparecerem, confira no site se a versão da sua ROM é suportada.

## Transferir Arquivos via Rede (NextUI)

Com Wi-Fi ativo, o NextUI permite transferir ROMs direto pelo computador, sem remover o cartão SD:

1. Acesse **Configurações → Rede** no NextUI e ative o servidor de arquivos (FTP ou Samba, conforme a versão).
2. No computador, abra o gerenciador de arquivos e conecte no endereço IP exibido no console.
3. Copie as ROMs para as pastas correspondentes.
4. Após a transferência, execute **Refresh ROMs** no menu do firmware.

## Ajustar Configurações de Emulação (RetroArch)

No KNULLI e muOS, o RetroArch é a base de emulação. Para ajustar:

<Steps>
  <Step title="Abra o Quick Menu durante o jogo">
    Use o atalho de hotkey do seu firmware para abrir o Quick Menu do RetroArch. No KNULLI/muOS o padrão é <kbd>SELECT</kbd> + <kbd>X</kbd>.
  </Step>
  <Step title="Troque o core se necessário">
    Em **Quick Menu → Options** você seleciona qual core (emulador) usar para o sistema atual. Para PS1, por exemplo, o Beetle PSX HW costuma ter melhor compatibilidade que o PCSX ReARMed em alguns jogos.
  </Step>
  <Step title="Ajuste a proporção de tela">
    A tela do Brick é 4:3 nativa. Em **Settings → Video → Scaling**, use **Core provided** ou **4:3** para evitar distorção.
  </Step>
  <Step title="Ative shaders (opcional)">
    Em **Quick Menu → Shaders**, shaders CRT ou Scanlines dão uma aparência de tubo sem custo de performance relevante nos sistemas 16-bit e anteriores.
  </Step>
  <Step title="Salve as configurações">
    Use **Save Core Override** para salvar para todos os jogos do sistema, ou **Save Game Override** só para o jogo atual.
  </Step>
</Steps>

## Organizar a Biblioteca com Arte (Scraping)

**NextUI**: instale o **Artwork Scraper** pela **Pak Store** (Configurações → Pak Store). Com Wi-Fi conectado, acesse o sistema desejado, pressione **MENU** e selecione a opção de scraping. A velocidade depende da conexão.

**MinUI**: arte manual — crie uma pasta `.res` dentro da pasta do sistema em `Roms/` e coloque imagens `.png` com o mesmo nome das ROMs. Trabalhoso, mas funciona offline.

Arte de caixa e mídia ocupa espaço relevante. Para uma biblioteca grande com arte completa, 128GB ou mais é o ideal.
