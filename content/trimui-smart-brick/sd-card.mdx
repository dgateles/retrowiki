O TrimUI Smart Brick tem **um único slot microSD**. O eMMC interno de 8GB é usado pelo sistema operacional de fábrica e não fica disponível como mídia de armazenamento — então o cartão SD é o único lugar para ROMs, BIOS e saves dos firmwares custom.

## Cartões Recomendados

| Marca    | Modelo             | Capacidade | Status                  |
| -------- | ------------------ | ---------- | ----------------------- |
| Samsung  | Pro Plus           | 64GB       | ✅ Recomendado          |
| SanDisk  | Extreme            | 128GB      | ✅ Recomendado          |
| Kodak    | —                  | 128GB      | ✅ Excelente            |
| Kingston | Canvas Go Plus     | 256GB      | ✅ Bom custo-benefício  |
| SanDisk  | Ultra              | 32GB       | ✅ Básico / OK          |
| Genérico | —                  | Qualquer   | ❌ Não recomendado      |

64GB é o mínimo prático. Com menos espaço você vai ficar gerenciando constantemente — e o NDS, que roda muito bem na tela de 400 PPI do Brick, gera ROMs de até 512MB cada.

<Alert type="warning" title="Cartões genéricos">
  Cartões sem marca conhecida são a causa número um de boot loop e corrupção de dados no TrimUI Smart Brick. Samsung, SanDisk e Kodak têm histórico sólido com o console.
</Alert>

## Sistema de Arquivos

Todos os firmwares disponíveis — NextUI, MinUI, muOS e KNULLI — trabalham com **FAT32** como sistema de arquivos padrão.

- Cartões **até 32GB**: o próprio sistema operacional formata em FAT32 sem problema.
- Cartões **acima de 32GB**: use o **Rufus** (Windows) ou o **SD Card Formatter** — o Windows nativo limita a formatação FAT32 a 32GB.

O NextUI aceita exFAT, mas cartões exFAT são mais suscetíveis a corrupção no Brick. Se der escolha, FAT32 é mais estável.

## Formatação Passo a Passo

### Windows

1. Baixe o [Rufus](https://rufus.ie/) (versão portátil, sem instalação).
2. Conecte o cartão SD com um leitor de qualidade.
3. Abra o Rufus e selecione o cartão no campo **Dispositivo**.
4. Defina **Seleção de boot** como `Não inicializável`.
5. Defina **Sistema de arquivos** como `FAT32`.
6. Clique em **Iniciar** e confirme o aviso de apagar dados.

### macOS

1. Abra o **Utilitário de Disco**.
2. Selecione o cartão SD na barra lateral.
3. Clique em **Apagar** e escolha o formato **MS-DOS (FAT)**.
4. Confirme a operação.

### Linux

```bash
# Substitua /dev/sdX pelo dispositivo correto
sudo mkfs.vfat -F 32 /dev/sdX1
```

## Estrutura de Pastas

Após instalar o firmware, a raiz do cartão SD deve conter:

| Pasta       | Conteúdo                                              |
| ----------- | ----------------------------------------------------- |
| `Roms/`     | Subpastas por sistema (ex: `Game Boy`, `PS`)          |
| `BIOS/`     | Arquivos de BIOS por sistema (ex: `PS/scph1001.bin`)  |
| `Saves/`    | Save states e saves in-game (gerenciado pelo firmware)|

No MinUI e NextUI, os nomes das subpastas dentro de `Roms/` precisam corresponder exatamente ao nome do sistema reconhecido pelo firmware. O NextUI também aceita um código entre parênteses para forçar o emulador — ex: `Nintendo 64 (N64)`, `Nintendo DS (NDS)`.

## Usando um Segundo Cartão SD

Como o Brick tem slot único, testar firmwares diferentes sem perder configurações é simples: grave cada firmware em um cartão SD separado e troque físicamente. Ao ligar o console, ele carrega o que estiver no slot. Sem risco de misturar configurações.
