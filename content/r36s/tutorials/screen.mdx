<Alert type="critical" title="Tela Preta após Atualização?">
  Se você atualizou o ArkOS e a tela ficou preta, não se desespere. O sistema
  está rodando, mas não consegue exibir imagem. Você precisa substituir o
  arquivo `.dtb` na partição BOOT.
</Alert>

## Como Corrigir (Tela Preta)

<Steps>
  <Step title="Acesse a Partição BOOT">
    Insira o cartão SD (TF1) no computador. Abra a partição chamada **BOOT**.
  </Step>

<Step title="Localize os DTBs">
  Geralmente, há uma pasta (ex: `BOOT/dtb` ou apenas arquivos na raiz) contendo
  variantes. O arquivo principal que o sistema lê é geralmente nomeado como
  `rk3326-r35s-linux.dtb` (no ArkOS).
</Step>

  <Step title="Substitua o Arquivo">
    Você precisará substituir o arquivo `rk3326-r35s-linux.dtb` por uma das variantes compatíveis com sua tela.
    
    *   Se você tem backup do cartão original, copie o arquivo `.dtb` dele para o novo cartão.
    *   Se não tem backup, tente os arquivos disponíveis nos repositórios da comunidade (AeolusUX).
  </Step>
</Steps>

## Correção de Gama (Panel 4)

Jogadores atentos podem ter notado que os valores de gama nas telas **Panel 4** podem estar desajustados, resultando em uma imagem "lavada" ou com uma tonalidade esverdeada. Existe uma correção via script que resolve isso, tornando o Panel 4 superior ao Panel 1 com cores muito mais vibrantes.

<Alert type="info">
  Essa correção é específica para melhorar a qualidade de imagem do **Panel 4**,
  mas também inclui valores para o Panel 1 (embora o efeito seja mais sutil).
</Alert>

### Instalação da Correção

<Steps>
  <Step title="Baixe o Arquivo">
    Faça o download do pacote de correção de gama no link abaixo.
  </Step>
  
  <Step title="Mova para o SD">
    Extraia o conteúdo e mova a pasta `Gamma/` para a pasta `tools/` na partição **EASYROMS** do seu cartão SD.
    
    O caminho final deve ser algo como: `EASYROMS/tools/Gamma/`.
  </Step>

  <Step title="Acesse no Console">
    No R36S, navegue até o menu **Options** > **Tools** > **Gamma**.
  </Step>

  <Step title="Ative a Correção">
    Selecione a opção **Enable Gamma Correction** para ativar a correção na inicialização (boot).
    
    *   Use **Set Panel1/4** para gravar os valores adequados para seu painel (só funciona se a correção estiver ativada).
    *   Use **Adjust Gamma (temporary)** para testar valores sem salvar (reseta ao reiniciar).
  </Step>
</Steps>
