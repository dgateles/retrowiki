## Console não Liga ou Trava na Logo

**Possíveis causas e soluções:**

1. **Bateria descarregada** — conecte o carregador por pelo menos 15 minutos antes de tentar ligar novamente
2. **Pressione e segure o botão Power por 10 segundos** para forçar o desligamento se estiver travado
3. **Reinicialização forçada** — pressione simultaneamente Power + Volume Baixo por 10 segundos
4. Se o problema persistir, acesse o **Recovery Mode**: Power desligado → segure Power + Volume Baixo → selecione "Wipe Cache Partition"

<Alert type="warning" title="Não Use Factory Reset como Primeira Opção">
  O Factory Reset apaga todos os dados do armazenamento interno, incluindo emuladores instalados e saves. Tente todas as outras opções antes.
</Alert>

## Superaquecimento / Desempenho Reduzido

O RG-406V tem **cooling ativo** (ventoinha interna), mas o UNISOC Tiger T820 pode esquentar em sessões longas com emuladores pesados.

**O que fazer:**

- Verifique se a ventoinha está funcionando — você deve ouvir um leve som ao rodar emuladores exigentes
- Ative o modo de gerenciamento de energia no Android: **Configurações → Bateria → Modo de Desempenho** — se o modo "Economia" estiver ativo, o desempenho é reduzido automaticamente
- Em sessões longas de PS2 ou GameCube, faça pausas ocasionais
- Não cubra as saídas de ar (parte traseira do console)
- Temperatura máxima segura de operação: até ~45°C na superfície traseira é normal

## Drift nos Analógicos (Hall Effect)

Os sticks Hall Effect do RG-406V raramente desenvolvem drift físico, mas podem apresentar descalibração de software.

**Sintoma:** personagem se move sozinho sem tocar no stick.

**Solução:**

1. **Calibração via Android**: Configurações → Sistema → Controladores (ou "Controlador de Jogo") → selecione o controle → Calibrar
2. **Calibração via RetroArch**: Configurações → Input → Port 1 Controls → Calibrar analógicos (deixe os sticks soltos durante o processo)
3. Se o problema ocorrer apenas em um jogo específico, verifique se o emulador tem configuração de dead zone própria
4. Em último caso: reset de configurações do controlador nas configurações Android

## Wi-Fi não Conecta ou Cai

O RG-406V usa **Wi-Fi 5 (802.11ac)** com dual band (2.4 GHz e 5 GHz).

**Problemas comuns:**

| Problema | Causa Provável | Solução |
|----------|---------------|---------|
| Não encontra redes | Wi-Fi desligado | Verifique a barra de status / painel rápido |
| Conecta mas sem internet | IP não atribuído | Esqueça a rede e reconecte |
| Cai frequentemente | Economia de energia | Veja abaixo |
| Velocidade baixa | Sinal fraco ou banda 2.4 GHz | Tente conectar à banda 5 GHz |

**Wi-Fi cai durante uso / sleep:**

O Android pode desativar o Wi-Fi em modo sleep para economizar bateria.

- **Configurações → Wi-Fi → Avançado → Wi-Fi em modo sleep**: selecione **Nunca desativar**
- Ou mantenha a tela ligada durante downloads

## Jogos com Baixo Desempenho / Quedas de FPS

**Para RetroArch:**

1. Verifique se o core correto está sendo usado (consulte a página de [Emuladores](/rg-406v/emuladores))
2. Ative **Vsync** se o jogo tremer: Configurações → Vídeo → Vsync
3. Para PS2 via RetroArch: troque para **NetherSX2 standalone** (desempenho muito melhor)
4. Para GameCube/Wii: troque para **Dolphin standalone** com backend Vulkan
5. Feche outros apps em segundo plano: botão Recentes → feche todos

**Para emuladores standalone:**

- Dolphin: troque de OpenGL para **Vulkan** nas configurações de vídeo
- NetherSX2: troque para **Vulkan** e reduza o upscaling de 4x para 2x
- PPSSPP: reduza a resolução interna se houver problemas (raro no RG-406V)

## Áudio com Delay ou Distorção

**Delay de áudio no RetroArch:**

1. Configurações → Áudio → Latência → reduza para 64ms ou menos
2. Se usar driver de áudio JAVA: troque para **OpenSL ES** ou **AAudio** (Android 8+)

**Distorção em volume alto:**

- Reduza o volume do sistema para ~70-80% e ajuste no emulador
- Os alto-falantes inferiores do RG-406V podem distorcer levemente no volume máximo

## Touchscreen não Responde Corretamente

- **Touchscreen lento**: verifique se a tela está com película muito grossa ou bolhas
- **Toque duplo/fantasma**: limpe a tela — digitais acumuladas podem causar toques indesejados
- **Calibração**: Android 13 não tem calibração nativa de touchscreen; se persistir, pode ser defeito de hardware

## Cartão SD não Reconhecido

1. Remova e reinsira o cartão com o console **desligado**
2. Verifique se o cartão está no formato exFAT ou FAT32 (formatos NTFS e ext4 não são reconhecidos nativamente)
3. Teste o cartão em um PC para verificar se está com defeito
4. Verifique se o slot está limpo — um sopro de ar comprimido pode ajudar

## Emulador Fecha Sozinho (Crash)

**Causas mais comuns no Android:**

1. **Arquivo de ROM corrompido** — baixe novamente ou verifique o checksum
2. **BIOS ausente ou incorreta** — especialmente para PS1, PS2 e Dreamcast
3. **Falta de memória** — feche outros apps antes de abrir emuladores pesados
4. **Versão desatualizada do emulador** — atualize pelo app ou baixe a versão mais recente

Para crashes no RetroArch especificamente:
- Apague o arquivo de configuração do core específico (não o global)
- Configurações → Atualizar Tudo → Update Assets + Update Core Info

## Botões Físicos sem Resposta

Se um ou mais botões pararem de responder:

1. Teste em **Configurações → Sistema → Teste de Botões** (se disponível no firmware)
2. Instale o app **Gamepad Tester** pela Play Store e verifique cada botão
3. Limpeza com ar comprimido nos botões pode resolver problemas de tecla presa
4. Se for defeito de fábrica (primeiros 30 dias), contate a Anbernic para garantia

<Alert type="info" title="Recuperação de Firmware">
  Se o Android corromper e o console não iniciar normalmente, o Recovery Mode permite limpar o cache ou executar um factory reset sem perder o sistema operacional. Acesse com Power + Volume Baixo mantidos durante o boot.
</Alert>
