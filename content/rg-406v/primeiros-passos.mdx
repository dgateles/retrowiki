O RG-406V chega de fábrica com **Android 13** e 128 GB de armazenamento UFS interno — você já tem espaço suficiente para uma boa biblioteca sem precisar de cartão SD imediatamente. O setup inicial é parecido com qualquer smartphone Android moderno, mas com algumas particularidades de um console de emulação.

## Configuração Inicial do Android

Ao ligar pela primeira vez:

1. Conecte ao Wi-Fi (Wi-Fi 5 está disponível — prefira redes 5 GHz para melhor velocidade)
2. Configure a conta Google se quiser acesso à Play Store
3. Ajuste data/hora e idioma em **Configurações → Sistema → Idioma**
4. Ative **Fontes Desconhecidas** em **Configurações → Segurança → Instalar apps desconhecidos** — necessário para APKs de emuladores fora da Play Store

<Alert type="info" title="GammaOS Next">
  O **GammaOS Next** é um firmware Android customizado desenvolvido especificamente para o RG-406V. Ele mantém o Android completo, mas remove bloatware da Anbernic e adiciona ajustes de performance e uma interface mais limpa para emulação. Disponível em retrocatalog.com — veja as instruções de instalação antes de proceder.
</Alert>

## Instalando Emuladores

### Via Play Store
Os seguintes emuladores estão disponíveis diretamente na Play Store:

- **RetroArch** — framework multi-sistema, base para a maioria dos usuários
- **PPSSPP** — PSP, melhor desempenho standalone
- **Dolphin** — GameCube e Wii
- **AetherSX2 / NetherSX2** — PlayStation 2 (NetherSX2 é um fork com melhorias)
- **Citra** — Nintendo 3DS (versão legacy ainda funcional)
- **M64Plus FZ** — Nintendo 64 standalone

### Via APK (Sideload)
Alguns emuladores importantes não estão mais na Play Store:

| Emulador | Sistema | Onde Baixar |
|----------|---------|-------------|
| Yuzu / Sudachi | Nintendo Switch | Repositório oficial do projeto |
| Lime3DS | Nintendo 3DS | GitHub do projeto |
| DuckStation | PlayStation 1 | GitHub |
| XEMU | Xbox Original | xemu.app |

## Adicionando ROMs

O RG-406V tem 128 GB internos — você pode organizar as ROMs diretamente no armazenamento interno ou em um cartão microSD.

Estrutura de pastas recomendada:
```
/storage/emulated/0/ROMs/
  ├── NES/
  ├── SNES/
  ├── GBA/
  ├── PS1/
  ├── PS2/
  ├── PSP/
  ├── N64/
  ├── GCN/
  └── ...
```

Transfira arquivos via:
- **USB-C** conectado ao PC (modo Transferência de Arquivos / MTP)
- **Wi-Fi** usando apps como Solid Explorer com FTP, ou ADB Wireless
- **Cartão microSD** formatado em exFAT

## BIOS Necessárias

Alguns sistemas exigem BIOS para funcionar corretamente. Coloque os arquivos de BIOS na pasta que cada emulador espera:

| Sistema | Arquivo de BIOS | Emulador |
|---------|----------------|----------|
| PlayStation 1 | `scph1001.bin` ou `scph5501.bin` | DuckStation / RetroArch |
| PlayStation 2 | `ps2-0230e-20080220.bin` (ou similar) | NetherSX2 |
| Dreamcast | `dc_boot.bin` + `dc_flash.bin` | Flycast / RetroArch |
| Saturn | `sega_101.bin` / `mpr-17933.bin` | Yaba Sanshiro / RetroArch |
| Game Boy Advance | `gba_bios.bin` | mGBA / RetroArch |

**No RetroArch**, as BIOS ficam em: `RetroArch/system/`

**No DuckStation**, configure o caminho em: *Configurações → Console → Diretório BIOS*

<Alert type="warning" title="Sobre BIOS">
  Por questões legais não fornecemos os arquivos de BIOS. Eles devem ser extraídos dos seus próprios consoles físicos. A maioria dos emuladores roda sem BIOS, mas com ela a compatibilidade e precisão melhoram significativamente — especialmente no PS1 e Dreamcast.
</Alert>

## Transferência via ADB (Avançado)

Para transferências mais rápidas via Wi-Fi, ative o **Depuração USB** em Configurações → Sobre o dispositivo → (toque 7x no número da versão para ativar as opções do desenvolvedor) → Opções do desenvolvedor → Depuração USB.

```bash
# Conectar via Wi-Fi (substitua pelo IP do console)
adb connect 192.168.1.XXX:5555

# Enviar ROMs para o console
adb push /caminho/local/roms/ /storage/emulated/0/ROMs/
```

## Primeiros Emuladores para Configurar

Se você está começando do zero, esta sequência faz sentido:

1. **RetroArch** — configure primeiro, pois vários outros emuladores usam seus cores
2. **PPSSPP** — PSP funciona melhor standalone; configure a resolução interna para 2x ou 3x
3. **NetherSX2** — PS2 exige configuração de BIOS e ajustes de backend de vídeo (Vulkan recomendado)
4. **Dolphin** — GameCube/Wii; use o backend Vulkan para melhor desempenho
5. **Lime3DS ou Citra** — 3DS standalone para melhor compatibilidade

Continue na página de [Emuladores e Frontends](/rg-406v/emuladores) para configurações específicas de cada sistema.
