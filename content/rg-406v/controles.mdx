## Layout dos Controles

O RG-406V usa um layout simétrico com os dois analógicos posicionados na mesma altura — diferente do layout assimétrico Xbox. O D-pad fica na posição **superior esquerda** (acima do analógico esquerdo).

<ButtonLayout screenLabel='IPS 4:3' centerTopLabel="HOME" />

### Botões Disponíveis

| Grupo | Botões |
|-------|--------|
| Face | A, B, X, Y |
| Ombro | L1, R1, L2, R2 |
| Analógicos | L3 (pressionar esquerdo), R3 (pressionar direito) |
| Direcionais | D-pad (cruz, posição superior) |
| Sistema | SELECT, START, HOME |
| Especiais | Volume +/-, Power |
| Sensores | Touchscreen, Giroscópio |

### Hall Effect Sticks

Os analógicos do RG-406V usam **tecnologia Hall Effect** — baseada em campo magnético, sem potenciômetros resistivos que se degradam com o tempo. O resultado prático:

- Sem drift por desgaste (o problema mais comum em controles tradicionais)
- Resposta mais precisa e consistente
- Vida útil significativamente maior

<Alert type="info" title="Calibração de Sticks">
  Mesmo com Hall Effect, os sticks podem precisar de calibração inicial. No Android: **Configurações → Controlador → Calibrar Analógicos**. No RetroArch: **Configurações → Input → Port 1 Controls → Calibrar Analógico**.
</Alert>

## Mapeamento no Android

O RG-406V usa o sistema de mapeamento padrão do Android para controles físicos. A maioria dos emuladores da Play Store reconhece os botões automaticamente.

### Mapeamento Padrão (Android)

| Botão Físico | Código Android |
|-------------|----------------|
| A | KEYCODE_BUTTON_A |
| B | KEYCODE_BUTTON_B |
| X | KEYCODE_BUTTON_X |
| Y | KEYCODE_BUTTON_Y |
| L1 | KEYCODE_BUTTON_L1 |
| R1 | KEYCODE_BUTTON_R1 |
| L2 | KEYCODE_BUTTON_L2 / AXIS_LTRIGGER |
| R2 | KEYCODE_BUTTON_R2 / AXIS_RTRIGGER |
| L3 | KEYCODE_BUTTON_THUMBL |
| R3 | KEYCODE_BUTTON_THUMBR |
| SELECT | KEYCODE_BUTTON_SELECT |
| START | KEYCODE_BUTTON_START |
| HOME | KEYCODE_BUTTON_MODE |

### Remapeamento via Android

Para remapear botões em nível de sistema operacional:

1. Vá em **Configurações → Sistema → Idioma e entrada → Físico**
2. Selecione o controlador do RG-406V
3. Use o mapeamento de teclas para reassociar funções

Alguns launchers e emuladores têm seu próprio sistema de remapeamento — verifique as configurações de cada app.

## Hotkeys no RetroArch

O RetroArch usa uma tecla de **hotkey** (por padrão o SELECT) combinada com outros botões para funções especiais.

### Hotkeys Padrão

| Combinação | Função |
|-----------|--------|
| SELECT + X | Abrir Menu RetroArch |
| SELECT + START | Fechar jogo (voltar ao menu) |
| SELECT + R1 | Salvar Estado (Save State) |
| SELECT + L1 | Carregar Estado (Load State) |
| SELECT + R2 | Avanço Rápido (Fast Forward) |
| SELECT + L2 | Retroceder (Rewind — se habilitado) |
| SELECT + B | Captura de Tela |
| SELECT + Y | Mostrar/Ocultar FPS |
| SELECT + A | Pausar emulação |
| SELECT + ↑ / ↓ | Trocar slot de Save State |

### Configurando a Tecla Hotkey

Se quiser mudar o botão de hotkey:

1. RetroArch → **Configurações → Input → Teclas de Atalho**
2. Selecione **Habilitar Tecla de Atalho** → pressione o botão desejado
3. As combinações de hotkey usarão o novo botão

### Hotkeys Específicas por Emulador Standalone

| Emulador | Ação | Combinação |
|----------|------|-----------|
| PPSSPP | Menu | HOME ou SELECT+START |
| PPSSPP | Salvar Estado | Configurável em Options → Controls |
| Dolphin | Menu | HOME |
| NetherSX2 | Pausar | START |
| Lime3DS | Menu | HOME |

## Configuração de Analógicos no RetroArch

Para sistemas que usam analógicos (N64, PS1, PS2, GameCube):

1. RetroArch → **Configurações → Input → Port 1 Controls**
2. **Tipo de Analógico**: selecione **Duplo Analógico** para PS1/PS2, ou **GameCube** para Dolphin core

### Dead Zone dos Analógicos

Se sentir movimento involuntário leve:

1. RetroArch → **Configurações → Input → Eixo de Morto do Analógico**
2. Aumente levemente o valor (padrão ~0.5, tente 0.1 a 0.15 para Hall Effect)

Os Hall Effect sticks geralmente precisam de dead zone menor que sticks tradicionais.

## Touchscreen e Giroscópio

O RG-406V tem **touchscreen** e **giroscópio** — recursos úteis em alguns cenários específicos:

**Touchscreen:**
- Navegação nos menus Android
- Jogos de DS que usam tela de toque (configure o melonDS ou DeSmuME para mapear a tela touch)
- Jogos Android nativos com controle por toque

**Giroscópio:**
- Jogos Wii que usam movimento (Dolphin suporta giroscópio via controle Android)
- Alguns jogos 3DS que usam sensor de movimento (Lime3DS)
- Para ativar no Dolphin: **Configurações → Controles → Wii Remote → Movimento → Usar Sensor do Dispositivo**

<Alert type="info" title="Botões de Volume como Atalhos">
  Os botões físicos de volume (+/-) funcionam como atalhos rápidos no Android. No sistema operacional, controlam o volume. Dentro de emuladores, você pode remapear para outras funções em apps que suportam remapeamento completo de botões.
</Alert>
