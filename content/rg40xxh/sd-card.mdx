| -------- | ------------------ | ---------- | --------------------------- |
| Samsung  | Pro Plus           | 64 GB      | Sistema (TF1)               |

Use cartões **Classe A1** no mínimo — A2 é melhor ainda. Velocidade de leitura baixa causa travamento no boot e demora excessiva ao carregar jogos.

## Dual Slot: TF1 para Sistema, TF2 para ROMs

O RG40XX H tem dois slots microSD: **TF1** (firmware/OS) e **TF2** (dados/ROMs). Usar os dois é a configuração padrão recomendada pelo KNULLI e pelo muOS, e faz sentido prático: ao atualizar o firmware no TF1, todo o conteúdo do TF2 — saves, estados, ROMs, box art — permanece intocado.

| Slot | Conteúdo     | Capacidade Sugerida |
| ---- | ------------ | ------------------- |
| TF1  | Firmware/OS  | 32 GB ou 64 GB      |
| TF2  | ROMs e saves | 128 GB ou 256 GB    |

### Configurar o TF2 no KNULLI

1. Insira o segundo cartão no slot **TF2** com o console desligado.
2. Ligue e acesse **Main Menu → System Settings → Storage**.
3. Em **Storage Device**, selecione o segundo cartão.
4. O sistema cria automaticamente a estrutura de pastas no TF2.

### Configurar o TF2 no muOS

No muOS, basta inserir o cartão no TF2 antes de ligar — o sistema detecta e usa automaticamente. As pastas ficam em `/MUOS/` no TF2.

**Localização dos saves:**
- muOS: saves em `/MUOS/save/file/`, estados em `/MUOS/save/state/`
- KNULLI (Batocera): saves em `/userdata/saves/`, estados em `/userdata/states/` — ambos no TF2 quando configurado

## Estrutura de Pastas (ROMs)

Após o boot inicial, a pasta de ROMs terá subpastas por sistema:

```
Roms/
├── GBA/        ← Game Boy Advance
├── GBC/        ← Game Boy Color
├── GB/         ← Game Boy
├── NES/        ← Nintendo NES
├── SNES/       ← Super Nintendo
├── MD/         ← Mega Drive / Genesis
├── PSX/        ← PlayStation 1
├── NDS/        ← Nintendo DS
├── PSP/        ← PSP
└── PORTS/      ← PortMaster
```

## Gravar o Firmware no Cartão

- **Windows**: [Rufus](https://rufus.ie/) ou [Balena Etcher](https://www.balena.io/etcher/)
- **macOS**: [Balena Etcher](https://www.balena.io/etcher/)
- **Linux**: `dd` ou Balena Etcher

A gravação apaga todos os dados do cartão. Se o Windows pedir para formatar o cartão depois de gravar, cancele — as partições Linux aparecem como não reconhecidas, e isso é normal.
