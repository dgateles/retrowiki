## Layout dos Botões

O Air X usa layout simétrico de analógicos — ambos os sticks ficam na posição central, alinhados horizontalmente, semelhante ao PlayStation. O D-Pad fica na posição superior esquerda.

<ButtonLayout screenLabel='IPS 5.5"' centerTopLabel="HOME" />

**Botões de ação (lado direito):** Y (cima-esquerda), X (cima-direita), B (baixo-esquerda), A (baixo-direita) — padrão Nintendo/Xbox.

**Triggers:** L2/R2 são triggers analógicos com curso completo, compatíveis com jogos que detectam pressão variável.

**Botões de sistema:** Home, Volume Up/Down, Menu (botão Android).

## Hall Effect Sticks

Os analógicos do Air X usam tecnologia Hall Effect magnética, sem contato físico entre as peças internas. Isso significa:

- Sem drift por desgaste ao longo do tempo
- Resposta mais precisa no centro
- Vida útil significativamente maior que sticks tradicionais

Não é necessário calibrar os sticks com frequência. Se notar drift, vá em **Configurações → Acessibilidade → Controles** e recalibre.

## Key Mapping no Android

O M Space (launcher nativo) já inclui um módulo de mapeamento de teclas. Para acessar:

1. Abra o **M Space**
2. Vá em **Configurações → Mapeamento de Teclas**
3. Selecione o perfil desejado ou crie um novo

Para apps Android que não reconhecem gamepad nativamente, use um app de mapeamento:

- **Mantis Gamepad Pro** (Play Store) — mapeia botões do gamepad para toques na tela
- **GameKeyboard** — alternativa gratuita

<Alert type="info" title="Gyroscope e Touch">
  O Air X tem giroscópio e touchscreen. Em emuladores que suportam gyro (PPSSPP, Citra/Lime3DS), ative nas configurações do emulador para jogos como WipEout Pure ou Kirby: Planet Robobot.
</Alert>

## Hotkeys do RetroArch

No RetroArch, configure um botão como **Hotkey Enable** para acionar os atalhos. O padrão recomendado é **Select**.

| Combinação | Função |
| ---------- | ------ |
| <kbd>Select</kbd> + <kbd>X</kbd> | Abrir menu RetroArch |
| <kbd>Select</kbd> + <kbd>R1</kbd> | Salvar estado (Save State) |
| <kbd>Select</kbd> + <kbd>L1</kbd> | Carregar estado (Load State) |
| <kbd>Select</kbd> + <kbd>R2</kbd> | Avanço rápido (Fast Forward) |
| <kbd>Select</kbd> + <kbd>L2</kbd> | Retroceder (Rewind, se habilitado) |
| <kbd>Select</kbd> + <kbd>B</kbd> | Captura de tela |
| <kbd>Select</kbd> + <kbd>A</kbd> | Pausar / Retomar |
| <kbd>Select</kbd> + <kbd>Start</kbd> | Sair do jogo (voltar ao menu) |
| <kbd>Select</kbd> + <kbd>Y</kbd> | Mostrar/ocultar FPS |

### Configurar os Hotkeys

1. RetroArch → **Settings → Input → Hotkeys**
2. Defina **Hotkey Enable Button** como Select
3. Configure cada ação conforme necessário

## Hotkeys PPSSPP (PSP)

| Combinação | Função |
| ---------- | ------ |
| <kbd>Home</kbd> (botão Android) | Abrir menu do PPSSPP |
| <kbd>Select</kbd> + <kbd>Start</kbd> | Menu do PPSSPP (in-game) |

Configure em PPSSPP → **Settings → Controls → Control Mapping**.

## Hotkeys Dolphin (GameCube/Wii)

| Ação | Configuração padrão |
| ---- | ------------------- |
| Abrir menu | <kbd>Menu</kbd> (botão Android) |
| Sair do jogo | Botão voltar ou menu Dolphin |

Para controles de Wii em jogos que usam Wiimote + Nunchuk, o Dolphin no Android permite mapear os movimentos ao giroscópio do console.

## Remapeamento por Jogo no RetroArch

Para jogos específicos que precisam de layout diferente:

1. Com o jogo aberto, pressione **Select + X** para abrir o menu
2. Vá em **Controls → Port 1 Controls**
3. Remapeie os botões como desejar
4. **Salve o remap**: Controls → **Save Game Remap File** (somente para esse jogo) ou **Save Content Directory Remap File** (para todos os jogos do sistema)

## Botões de Volume e Brilho

| Ação | Método |
| ---- | ------ |
| Volume | Botões físicos de volume (lateral do console) |
| Brilho | Notificação rápida do Android (deslize de cima para baixo) |
| Brilho rápido (GammaOS Next) | Botão M + D-Pad vertical (varia por versão do firmware) |
