O Air X chega pronto para uso: Android 14 instalado, emuladores pré-configurados e o launcher M Space já ativo. O que você precisa fazer é organizar suas ROMs e ajustar algumas configurações de sistema.

## Primeiro Boot

Ao ligar o console pela primeira vez:

1. Conecte ao Wi-Fi (configurações do Android → Wi-Fi)
2. Opcionalmente, faça login na Google Play Store para instalar apps adicionais
3. Verifique atualizações de sistema em **Configurações → Sistema → Atualização**
4. Insira um cartão microSD para armazenar ROMs (veja a seção [Cartão SD](/mangmi-air-x/sd-card))

<Alert type="info" title="Emuladores pré-instalados">
  O Air X já vem com uma seleção de emuladores configurados de fábrica. Você pode complementar com versões mais recentes via Google Play ou APKs, mas o essencial já está pronto.
</Alert>

## Organização das ROMs

O método mais prático é seguir a estrutura de pastas do ES-DE, compatível com Daijisho e outros frontends:

```
/sdcard/ROMs/
  nes/
  snes/
  gba/
  gbc/
  gb/
  genesis/
  ps1/
  psp/
  nds/
  dreamcast/
  n64/
  saturn/
  gamecube/
```

Coloque as ROMs nos subdiretórios correspondentes ao sistema. Ao abrir o frontend, ele escaneará automaticamente e exibirá sua biblioteca.

### Transferência de Arquivos

Há três formas de transferir ROMs para o console:

| Método | Velocidade | Facilidade |
| ------ | ---------- | ---------- |
| Cabo USB (MTP) | Alta | Fácil — conecte o console ao PC, acesse como armazenamento |
| Wi-Fi via FTP (app ES File Explorer ou similar) | Média | Requer app FTP no console |
| Leitor de cartão SD (PC) | Muito alta | Retire o SD, insira no PC, copie direto |

O método mais rápido para grandes bibliotecas é o leitor de cartão no PC.

## BIOS Necessárias

Alguns emuladores exigem arquivos de BIOS para funcionar corretamente. Esses arquivos devem ser obtidos legalmente a partir das suas próprias cópias de hardware.

| Sistema | Arquivo de BIOS | Pasta destino |
| ------- | --------------- | ------------- |
| PlayStation 1 | `scph1001.bin`, `scph5501.bin` | `/RetroArch/system/` |
| Dreamcast | `dc_boot.bin`, `dc_flash.bin` | `/RetroArch/system/dc/` |
| Nintendo DS | `bios7.bin`, `bios9.bin`, `firmware.bin` | `/RetroArch/system/` |
| Saturn | `sega_101.bin` ou `mpr-17933.bin` | `/RetroArch/system/` |
| GBA | `gba_bios.bin` | `/RetroArch/system/` (opcional, mas recomendado) |

<Alert type="warning" title="BIOS e legalidade">
  Arquivos de BIOS são protegidos por direitos autorais. Faça dump a partir do seu próprio hardware ou pesquise sobre as alternativas de BIOS livre disponíveis para cada sistema.
</Alert>

## Configurações Recomendadas do Android

Ajustes úteis logo após o primeiro uso:

### Desempenho
- **Configurações → Bateria → Modo de desempenho**: ative quando for jogar títulos mais pesados (GameCube, PS2)
- **Configurações → Desenvolvedor → Escala de animação**: reduza para 0.5x para navegação mais ágil

### Ativar Opções do Desenvolvedor
1. Vá em **Configurações → Sobre o dispositivo**
2. Toque em **Número da versão** sete vezes
3. Volte para Configurações — o menu **Opções do Desenvolvedor** estará disponível

### Armazenamento
O Air X vem com 64 GB de eMMC interno. Para uma biblioteca de jogos maior, um cartão microSD de 256 GB ou mais é fortemente recomendado.

## Firmware Alternativo (opcional)

Se quiser uma experiência mais focada em emulação, o Air X suporta dois CFWs:

- **GammaOS Next** — Android 14 otimizado para handheld gaming, com e sem Google Apps. Instalação via SD card. [Guia oficial no retrohandhelds.gg](https://retrohandhelds.gg/how-to-install-gammaos-next-mangmi-air-x/)
- **ROCKNIX** — distribuição Linux dedicada à emulação. [Página oficial](https://rocknix.org/devices/mangmi/air-x/)

O Android de fábrica é suficiente para a maioria dos usuários. CFW é para quem quer personalização avançada ou desempenho extra.
